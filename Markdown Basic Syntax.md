옵시디언은 마크다운 문법을 사용하여 문서를 작성한다. 
일반적인 텍스트는 워드나 한글 같이 단순히 글자만 작성하면 된다!
<br>
> [!idea] 마크다운이란?
> **정의**
> 마크다운은 텍스트 기반의 마크업 언어로, 2004년 존 그루버가 만들었다. 
> HTML 언어로의 변환도 가능하며, 쉽게 쓰고 읽을 수 있어 블로그나 노트 앱에서 많이  활용되고 있다.
> <br>
> **장단점**
> - 마크다운은 간결하고, 변도의 도구 없이 작성할  수 있다.
> - 또한 텍스트 파일 형태로 저장되어 차지하는 용량이 작다.
> - 지원하는 프로그램과 플랫폼이 다양하지만, 표준이 없어 도구마다 변환 방식이 다르다.
> - 그리고 모든 HTML 마크업을 대신하지 못한다.
> - HTML 태그를 활용할 경우 해당 태그 내에서 마크다운 문법을 혼용할 수 없다.

## 제목(Headings)
- 문단을 형성하거나 주제를 환기할 때 적합하다.
- 마크다운에서는 # 문자를 통해 제목을 형성할 수 있다.
	- 마크다운에서 제공하는 제목은 **글자 크기 키움**과 **bold** 처리가 들어가있다.
- 제목은 **# 의 개수**로 결정된다. 
	- `#` 의 개수가 많아질수록 크기가 작아진다!
- 1-6개까지 적용할 수 있는데 사용 예는 아래와 같다.


| Markdown        | HTML               | Rendered Output  |
| --------------- | ------------------ | ---------------- |
| `# Title 1`     | `<h1>Title 1</h1>` | <h1>Title 1</h1> |
| `## Title 2`    | `<h2>Title 2</h2>` | <h2>Title 2</h2> |
| `### Title 3`   | `<h3>Title 3</h3>` | <h3>Title 3</h3> |
| `#### Title 4`  | `<h4>Title 4</h4>` | <h4>Title 4</h4> |
| `##### Title 5` | `<h5>Title 5</h5>` | <h5>Title 5</h5> |
| `###### Title 6`                |  `<h6>Title 6</h6>`                  | <h6>Title 6</h6>                 |

<br>

> [!note] 
> 몰랐는데 **-** 과 **=** 으로도 제목을 만들 수 있었다!!
> 
| Markdown         | HTML               | Rendered Output  |
| ---------------- | ------------------ | ---------------- |
| `Title 1 ========` | `<h1>Title 1</h1>` | <h1>Title 1</h1> |
| `Title 2 --------`                 |  `<h2>Title 2</h2>`                  |          <h2>Title 2</h2>        |
> 

<br>


> [!caution] Best Practice
> 1. 제목 표시 후에는 띄워쓰기를 하자
> 
| ✅ Do this            | ⛔️ Don't do this       |
| ------------------ | ------------------- |
| # Here's a Heading | `#Here's a Heading` |
> 2. 제목을 사용할 때는 빈 라인을 추가하자
> 
| ✅ Do this                                 | ⛔️ Don't do this       |
| --------------------------------------- | ------------------- |
| 제목쓰기 전에 이줄 비워두자아<br><br> `# Title`<br><br> 제목 쓰고 나서도 비워주기 | 빈 줄 없이 사용한다면 안돼요<br> `# Title`<br> 이렇게 쓰는건 좋지 않음 |
|                                         |                     |

## 문단(Paragraphs)
- 문단의 컨텐츠를 형성하고 문단을 구분짓기 위해서는 빈 줄을 통해 구분할 수 있다.

| Markdown | HTML | Rendered Output |
| -------- | ---- | --------------- |
|   안녕하세요!<br><br> 마크다운에 대해 배워봅시다:)       |  `<p>안녕하세요!</p>`<br> <br> `<p>마크다운에 대해 배워봅시다:)`    | <p>안녕하세요!</p> <p>마크다운에 대해 배워봅시다:)</p>                 |

<br>

> [!caution] Best Practice
> 리스트 형태가 아닌 경우에는 공백이나 탭을 주지 말 것!
> 
> | ✅ Do this            | ⛔️ Don't do this       |
| ------------------ | ------------------- |
| 문단을 쓸 때는 공백이나 탭을 사용하지 않아요<br><br>이렇게 바로 문장을 작성하세요! | &nbsp;&nbsp;&nbsp;&nbsp;이렇게 공백을 넣어버리면 문법에 맞지 않다고 해요.<br><br>왜그런지는 잘모르겠습니다. |

## 줄바꿈(Line Break)
- 마크다운에서는 아무리 엔터를 눌러도 엔터 개수만큼 줄이 띄워지지 않는다.     
- 이를 위해서는 HTML의 **`<br>`** 태그를 활용해야 한다! 

| Usage | Rendered Output |
| ----- | --------------- |
| `첫 번째 줄입니다.<br>`<br> `두 번째 줄입니다.`      |  첫 번째 줄입니다.<br> 두 번째 줄입니다.               |

> [!caution] `<br>`  태그 사용 시 유의할 점
> `<br>` 태그 작성 후 바로 엔터치고 글을 작성하면 줄바꿈이 적용되지 않는다. 반드시 엔터를 2번 칠 것! 
> ```html
> 예)
> 	<br>
> 	
> 	Hello World!
>```
> 그리고 **백슬래시(\)**는 **CommaonMark** 등 몇몇 애플리케이션에서만 지원하고 있어서 `<br>` 태그를 이용하여 줄바꿈하는 것을 추천한다고 함!

## 글자 강조 표현(Emphasis)
- 글에 대한 강조가 필요할 땐 굵게, 이탈릭체, 밑줄, 취소선을 활용해볼 수 있다.

<br>

### Bold
-  영단어 그대로 글자를 굵게 표시하여 강조한다.
	- 별표 두개(`**`)와 밑줄 두개(`__`), 그리고 **Ctrl+B** 단축키를 사용하여 적용할 수 있다.
<br>

| Markdown                        | HTML                                         | Rendered Output               |
| ------------------------------- | -------------------------------------------- | ----------------------------- |
| `글자를 **굵게 강조** 해보아요` | `글자를 <strong>굵게 강조</strong> 해보아요` | 글자를 **굵게 강조** 해보아요 |
| `글자를 __굵게 강조__ 해보아요` | `글자를 <strong>굵게 강조</strong> 해보아요` | 글자를 **굵게 강조** 해보아요 |
<br>

>[!caution] Best Practice
> 단어의 일부 글자를 강조하고자 한다면 띄워쓰기 없이 강조 구문을 사용한다. 
> 
> | Method 1                  | Method 2 | Rendered Output |
| ------------------ | ------------------- | --|
|     `단어**강조**하기`               |       `단어__강조__하기`              | 단어**강조**하기

<br>

### Italic
- 텍스트를 이탤릭체로 표시할 수 있다. (기울이기 효과)
- bold와 마찬가지로 별표와 밑줄을 이용하지만, 하나씩만 입력하면 된다.
	- 단축키는 **Ctrl+I**이다. 

| Markdown                          | HTML                                    | Rendered Output                       |
| --------------------------------- | --------------------------------------- | ------------------------------------- |
| `글자를 *기울여서* 강조해보아요`  | `글자를 <em>기울여서</em> 강조해보아요` | 글자를 <em>기울여서</em> 강조해보아요 |
| `글자를 _기울여서_ 강조해보아요_` | `글자를 <em>기울여서</em> 강조해보아요` | 글자를 <em>기울여서</em> 강조해보아요 |                                         |                                       |>[!caution] Best Practice

<br>

> [!caution] Best Practice
>단어의 일부 글자를 강조하고자 한다면 띄워쓰기 없이 강조 구문을 사용한다. 
> 
> | Method 1                  | Method 2 | Rendered Output |
| ------------------ | ------------------- | --|
|     `단어*강조*하기`               |       `단어_강조_하기`              | 단어*강조*하기|

<br>

>[!file] Bold와 Italic 함께 사용하기
>

| ------------------------------- | ------------------------------- | --------------- |

### Under line
- 밑줄은 HTML 태그를 이용하여 적용할 수 있다.
- 마크다운에서 밑줄과 관련된 표현은 `__` 기호라고 종종 나오긴 하나, 여기서는 강조 문법으로 사용되어 HTML 태그로 사용해야 할 것 같다.

| HTML | Rendered Output |
| ---- | --------------- |
| `<u>밑줄</u>을 그어 강조해보아요`     |   <u>밑줄</u>을 그어 강조해보아요              |

<br>

### 취소선
- 취소선은 물결 2개(`~~`)로 표현할 수 있다.

| Markdown                         | HTML                                    | Rendered Output                       |
| -------------------------------- | --------------------------------------- | ------------------------------------- |
| `~~취소선~~을 통해 강조해보아요` | `<del>취소선</del>을 통해 강조해보아요` | <del>취소선</del>을 통해 강조해보아요 |
|     `~~취소선~~을 통해 강조해보아요`                             |   `<s>취소선</s>을 통해 강조해보아요`                                      |         <s>취소선</s>을 통해 강조해보아요                              |

## 인용(Blockquotes)
- 마크다운에서 중요한 문단을 강조하거나 인용구를 삽입할 때 유용한 요소가 있다!
	- 인용구와 콜아웃이다(개인적으로 콜아웃 사랑함❤️)
- 인용구와 콜아웃은 `>` 기호를 통해 만들어볼 수 있다.
	- 몰랐는데 HTML 태그도 있었다!!! 마크다운 문법 정리하면서 HTML 태그도 하나씩 알아갈 수 있어서 좋구만!
- 여러 문장을 이용하는 것도 가능하고 중첩된 표현도 할 수 있다.

| Markdown                   | HTML                                              | Rendered Output                                 |
| -------------------------- | ------------------------------------------------- | ----------------------------------------------- |
| `> 이 문장은 인용되었어요` | `<blockquote>이 문장은 인용되었어요</blockquote>` | <blockquote>이 문장은 인용되었어요</blockquote> |
| `> 이 문장은 인용되었어요`<br>`>`<br> `>> 또 인용이 있어요!`                           | `<blockquote>이 문장은 인용되었어요 <blockquote>또 인용이 있어요!</blockquote></blockquote>`                                                  |                   <blockquote>이 문장은 인용되었어요 <blockquote>또 인용이 있어요!</blockquote></blockquote>                              |


<br>

> [!note] 다른 요소들과 함께 사용하기
> - 인용구는 마크다운의 다른 element들과 함께 사용할 수 있다. 
> 
| Markdown                                                  | Rendered Output |
| --------------------------------------------------------- | --------------- |
| `##### 인용구의 제목이에요`<br> `- 리스트도 넣어볼 수 있어요`<br>`- 여러 개 넣어도 괜찮아요!`<br><br>`글자를 *기울이거나* **굵게** 표현할 수도 있어요` |      <h5>인용구의 제목이에요</h5><br><li>리스트도 넣어볼 수 있어요</li><li>여러 개 넣어도 괜찮아요!</li><br><br>글자를 *기울이거나* **굵게** 표현할 수도 있어요             |   


<br>

>[!caution] Best Practice
> 인용구를 사용할 때는 앞, 뒤 문장에 빈 줄을 추가하자!
>  
>  | ✅ Do this            | ⛔️ Don't do this       |
| ------------------ | ------------------- |
| 인용구를 사용할 때는 라인을 추가해줘야 해요<br><br> > 여기는 인용구를 작성해요<br><br>인용구가 끝난 다음줄에도 추가해줘요| 이렇게 붙여써버리면 안돼요<br>> 여기는 인용구에요<br>음 근데 테스트 해보니까 라인 추가안해도 결과는 잘나오긴하네요 |

## 리스트(List)
- HTML과 마찬가지로 순서가 있고 없는 리스트에 대해 모두 다 작성할 수 있다.
- 순서가 있는 리스트는 `숫자.` 형태로 작성하고, 순서가 없는 리스트는 `-` 형태로 사용한다.

<br>

### 순서가 있는 리스트(Ordered List)
- 작성한 숫자를 시작점으로 삼는다.
	- 첫 번째 요소를 작성한 후 엔터를 입력하면 다음 숫자 리스트가 자동적으로 생성된다.
- 순서를 정확하게 맞춰적지 않더라도 자동적으로 정렬해준다.


| Markdown                                                                                                 | HTML                                                                                                                                                             | Rendered Output                                                                                          |
| -------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| 1. 첫 번째 아이템이에요<br>2. 두 번째 아이템이에요<br>3. 세 번째 아이템이에요<br>4. 네 번째 아이템이에요 | `<ol>`<br>`<li>첫 번째 아이템이에요</li>`<br>`<li>두 번째 아이템이에요</li>`<br> `<li>세 번째 아이템이에요</li>`<br> `<li>네 번째 아이템이에요</li>`<br> `</ol>` | 1. 첫 번째 아이템이에요<br>2. 두 번째 아이템이에요<br>3. 세 번째 아이템이에요<br>4. 네 번째 아이템이에요 |
| 1. 첫 번째 아이템이에요<br>1. 두 번째 아이템이에요<br>1. 세 번째 아이템이에요<br>1. 네 번째 아이템이에요 | `<ol>`<br>`<li>첫 번째 아이템이에요</li>`<br>`<li>두 번째 아이템이에요</li>`<br> `<li>세 번째 아이템이에요</li>`<br> `<li>네 번째 아이템이에요</li>`<br> `</ol>` | 1. 첫 번째 아이템이에요<br>2. 두 번째 아이템이에요<br>3. 세 번째 아이템이에요<br>4. 네 번째 아이템이에요 |
| 1. 첫 번째 아이템이에요<br>8. 두 번째 아이템이에요<br>3. 세 번째 아이템이에요<br>5. 네 번째 아이템이에요 | `<ol>`<br>`<li>첫 번째 아이템이에요</li>`<br>`<li>두 번째 아이템이에요</li>`<br> `<li>세 번째 아이템이에요</li>`<br> `<li>네 번째 아이템이에요</li>`<br> `</ol>` | 1. 첫 번째 아이템이에요<br>2. 두 번째 아이템이에요<br>3. 세 번째 아이템이에요<br>4. 네 번째 아이템이에요 |                                                                                                         |                                                                                                                                                                  |                                                                                                          |

<br>

>[!note] 하위 목록을 가진 리스트
> 하위 목록을 만들기 위해서는 엔터 후에 단순히 Tab 키를 누르면 된다.
> 1. 첫 번째입니다.
> 	1. 일 다시 일이라고 할 수 있겠어요.
> 	2. 일 다시 이라고 할 수 있겠어요.
> 2. 두 번째입니다.
> 
> HTML로 작성해보면 아래와 같다.
> ``` html
> <ol>
> 	<li>첫 번째입니다.</li>
> 		<ol>
> 			<li>일 다시 일이라고 할 수 있겠어요.</li>
> 			<li>일 다시 이라고 할 수 있겠어요.</li>
> 		</ol>
> 	<li>두 번째입니다.</li>
> </ol>
> ```


> [!note] 연도. 형태를 사용한 리스트
> - `연도.`을 입력하면 시작점을 기준으로 순서있는 리스트가 생성된다.
> - 그냥 단순히 `연도.` 형태를 작성하고 싶다면 `연도\.` 형태로 `.`을 escape 해줘야 한다.
> 
 | Year List | Escape dot |
 | -------- | --------------- |
 | 2023. 올 해는 2023년입니다.<br>2024. 내년은 2024년 입니다.         |        `2023\. 올 해는 2023년입니다.`<br>`2024\. 내년은 2024년 입니다.`          |


>[!caution] Best Practice
> - `숫자.` 형태로 작성하기. `숫자)` 형태도 작성할 수 있지만, 모든 애플리케이션에서 지원하는 것은 아니라고 한다.

<br>

## 링크(Link)
### 외부 링크
- 링크를 생성하기 위해서는 하이퍼링크가 될 텍스트를 대괄호로 감싸고, 바로 뒤에 이어서 실제 링크를 소괄호로 묶어주면 된다.

| Markdown | Rendered Output |
| -------- | --------------- |
|   `[유튜브](https://www.youtube.com) 영상 시청`       |  [유튜브](https://www.youtube.com) 영상 시청               |

<br>

### 내부 링크 
- 페이지 내에서 링크를 걸고 싶다면 **Heading IDs**를 활용할 수 있다. 
-> 테스트 해보니 옵시디언에서는 안먹힌다 🥲

>[!idea] Heading ID
> - 여러 마크다운 프로세서에서 제목에 대한 사용자 정의 ID를 제공한다.
> 	- 일부 마크다운 프로세서에서는 자동으로 이를 추가한다고 한다.
> - 사용자 정의 ID를 추가하면 제목에 링크를 걸어 연결하고 CSS를 활용하여 제목을 수정할 수 있다.
> 	- HTML에서 사용하는 ID와 동일하다고 보면 된다!
> - 중괄호와 `#`기호를 활용하여 생성할 수 있다.
> 
| Markdown | HTML |
| -------- | ---- |
| `### 제목입니다 { #custom-id }`         |   `<h3 id="custom-id"> 제목입니다</h3>`   |

- Heading ID를 생성하였다면 다음과 같이 사용할 수 있다.

| Markdown | HTML | Rendered Output |
| -------- | ---- | --------------- |
| `[Heading IDs](#heading-ids)`         |   `<a href="#heading-ids">Heading IDs</a>`   |   [Heading IDs](#heading-ids)              |

- 만약 새 탭이나 창으로 열고 싶다면, HTML을 활용해야 한다.
	- **target="_blank"** 속성을 활용하면 하이퍼링크 시, 새 창으로 열린다!
```html
	<a href="#heading-ids" target="_blank">Heading IDs</a>
```

<br>

> [!note] class로 링크 생성하기
> - 프캠에서 audit report 작성했을 때, 클래스에 대해서 링크를 걸었을 때에도 잘 동작했던 것 같은데 옵시디언에서는 안먹히는 것 같다..!

### 링크 효과
#### 링크 제목 출력
- 링크에 마우스를 가져다 대었을 때 제목이 출력되도록 할 수 있다.
- URL 뒤에 공백을 한 칸 주고 출력될 제목 텍스트를 입력해주면 된다.

| Markdown | Rendered Output |
| -------- | --------------- |
|   `[유튜브](https://www.youtube.com "유튜브 바로가기") 영상 시청`       |  [유튜브](https://www.youtube.com "유튜브 바로가기") 영상 시청               |

<br> 

#### 빠른 링크 생성
- 단순히 링크 그대로 보이고 하이퍼링크를 걸고 싶다면 **< >** 기호를 활용할 수 있다.(이메일도 가능)

| Markdown                   | Rendered Output          |
| -------------------------- | ------------------------ |
| `<https://www.google.com>` | <https://www.google.com> |
| `<sikk@naver.com>`           |     <sikk@naver.com>                     |

<br>
